Fixing Windows Time Sync Issue: "No time data was available"

Published July 31, 2025

Encountering the "The computer did not resync because no time data was available" error can be frustrating. This simple guide will walk you through the steps to fix the issue by adjusting a setting in the Registry Editor and then forcing a time synchronization.

Step 1: Open the Registry Editor

Press `Windows + R`, type `regedit`, and press Enter. This will open the Registry Editor. If a User Account Control (UAC) prompt appears, click 'Yes'.

Step 2: Navigate to the W32Time Parameters

In the Registry Editor, navigate to the following path:

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\W32Time\Parameters

Step 3: Modify the Type Value to NTP

On the right-hand side, find the `Type` entry. Double-click on it to open the "Edit String" dialog box. Change the `Value data` to `NTP` and click 'OK'.

Step 4: Force a New Sync via Command Prompt

After modifying the registry, you need to force a time synchronization. Open the Command Prompt as an administrator (Search for `cmd`, right-click, and 'Run as administrator').

Type the following command and press Enter:

w32tm /resync

You should see a message confirming that the resync was successful.

Your Windows machine should now be properly synchronized, and the time sync error should be resolved.
